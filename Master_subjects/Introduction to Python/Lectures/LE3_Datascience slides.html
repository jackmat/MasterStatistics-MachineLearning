NumPy¶

  • Built around the ndarray datatype.
    • nn-dimensional typed array
    • Contains homogenous data of a specific type
    • Vectorized operations
    • Implemented in C, i.e. fast
  • Contains lots of legacy code for backwards compatibility reasons
    • Anything outside arithmetic operations have SciPy versions
    • Use SciPy
  • Caveats:
    • NumPy has a matrix datatype specifically for 2d-arrays. AVOID
    • The matrix datatype is less general than ndarray, behaves differently and is rarely returned from built-in functions in NumPy or SciPy.
    • NumPy is more similar to R than Python in general but there are still differences to snag on

  • The ndarray datatype implements most standard operators for arrays
  • As the data is nn-dimensional, default multiplication is element-wise, not matrix multiplication
  • The dot function performs vector or matrix multiplication
  • The transpose instance function provides the transpose of any array

SciPy¶

  • Lots of numerical algorithms
  • Sub-modules for domain specific algorithms in areas such as:
    • Linear algebra
    • Statistics
    • Optimization
    • Signal processing
    • Sparse matrices
    • Etc.
  • Built on top of NumPy and its ndarray class
  • Most data and computational science libraries for Python are built on top of SciPy and NumPy or compatible with them

Pandas¶

  • Provides Python much of the data wrangling which makes R so popular for data science
  • DataFrame class similar to the R dataframe with fancy indexing
  • Series class for time series
  • SQL style merging of DataFrame objects
  • File readers for many popular file formats (csv, Excel, SAS
  • Plotting in conjuction with matplotlib

Matplotlib¶

  • Mainly 2D plots (some 3D plots)
  • Heavily inspired by MATLAB
  • Submodule pyplot provides state-machine like plotting
    • Similar to R's standard plot functions
    • Suitable for quick generation of simpler plots
  • Submodule axes provides an object-oriented API to matplotlib
    • Similar to ggplot library for R
    • Suitable for more advanced plots or where more control is wanted

SciKit-learn¶

  • Machine Learning for Python
  • Implements most established ML algorithms and new ones are added regularly
  • "Weka for Python"
  • Highly tweakable
  • Classification, Regression and Clustering
  • Dimensionality Reduction, Model Selection and Preprocessing
  • Includes the most common toy datasets for experimenting (Iris, Boston house prices, a reduced version of MNIST, etc.)
In [79]:
from sklearn import datasets
from sklearn.model_selection import cross_val_predict
from sklearn import linear_model
import matplotlib.pyplot as plt

lr = linear_model.LinearRegression()
boston = datasets.load_boston()
y = boston.target

# cross_val_predict returns an array of the same size as `y` where each entry
# is a prediction obtained by cross validation:
predicted = cross_val_predict(lr, boston.data, y, cv=10)

fig, ax = plt.subplots()
ax.scatter(y, predicted, edgecolors=(0, 0, 0))
ax.plot([y.min(), y.max()], [y.min(), y.max()], 'k--', lw=4)
ax.set_xlabel('Measured')
ax.set_ylabel('Predicted')
plt.show()

Honorable mentions¶

Keras¶

Neural networks API which can work on top of either Theano or TensorFlow. Very nice but can be a bit tricky to install, especially if you want to do computations on a GPU.

NLTK¶

If you want to do Natural Language Processing in Python, NLTK is the traditionally safe choice. However, there seems to be quite a few NLP programmers who are discontent with developments regarding NLTK the last few years. This means that it is not improbable that there might be a fork in the near future with NLTK splitting and continuing along two different lines.

Gensim¶

Library to work with vector space models, like distributional semantics (topic models, word embeddings, etc.) or recommender systems.

Scrapy¶

Want to collect web data? Scrapy is a library to build spiders and web crawlers.

Statsmodels¶

The Statsmodels library supplies a large chunk of the statisticians toolbox implementing traditional descriptive statistics and many standard tests and measures.

SymPy¶

As there is a NumPy for numerical mathematics there is a SymPy for symbolic mathematics. Generally, computational science tends to be heavy on the numerical side but there are instances where symbolic math shines, especially where numerical equivalencies tend towards large precision errors.

Bokeh¶

A package for making interactive vizualisations for the web. Similar to R's Shiny apps.

Seaborn¶

Built on top of matplotlib and provides conveniency implementations for statistical vizualisations. "If matplotlib tries to make easy things easy and hard things possible, seaborn tries to make a well-defined set of hard things easy too." - Michael Waskom
